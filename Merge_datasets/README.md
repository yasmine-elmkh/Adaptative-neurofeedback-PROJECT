# Merge_datasets – Fusion des données EEG Concentration vs Stress

Ce document décrit le script `merge_datasets.py` qui fusionne deux jeux de données EEG bruts (Cognitive Load pour la concentration, SAM40 pour le stress) en un seul jeu d’époques prétraitées, prêt pour l’augmentation et l’apprentissage.

---

## English Version

### Overview

The script `merge_datasets.py` loads, preprocesses, segments, and merges two EEG datasets (channel Fp2, sampling rate harmonized to 250 Hz, 4‑second epochs with 50% overlap). It produces three NumPy files (`X_merged.npy`, `y_merged.npy`, `subject_ids_merged.npy`) that can be used by the subsequent augmentation and classification pipelines.

### Folders and Files

- **`Merge_datasets/datasets_merged/`** (created automatically)
  - `X_merged.npy` : array of shape `(n_epochs, 1000)` – preprocessed and z‑score normalised epochs.
  - `y_merged.npy` : labels (`0` = Concentration, `1` = Stress).
  - `subject_ids_merged.npy` : subject IDs (`0-14` for Concentration, `15-54` for Stress after offset).

- **`Merge_datasets/merge_datasets.py`** – the script itself.

- **`outputs_validation/`** (optional, generated by a separate validation script)
  - Contains e.g. `merge_validation.png`, `README.md` for validation.

- **Input data paths** (expected before running):
  - `Dataset/Cognitive Load Assessment Concentration/raw_data/Arithmetic_Data/highlevel-*.txt`
  - `Dataset/Cognitive Load Assessment Concentration/raw_data/Stroop_Data/highlevel-*.txt`
  - `Dataset/Stress_dataset/raw_data/*Arithmetic_sub_*_trial*.mat`
  - `Dataset/Stress_dataset/scales.xls`

### Methodology

#### Common Preprocessing (per epoch)
1. High‑pass Butterworth (4th order, cutoff 0.5 Hz) – removes drift.
2. Low‑pass Butterworth (4th order, cutoff 40 Hz) – keeps EEG bands up to low gamma.
3. Notch filter at 50 Hz – removes power line noise.
4. Z‑score normalisation `(x - μ)/σ` – makes epochs comparable.

#### Segmentation
- Epoch duration : 4 seconds (`EPOCH_S = 4.0`).
- Overlap : 50% (`OVERLAP = 0.5`).
- Target sampling rate : 250 Hz (`FS_TARGET = 250`).
  - **Cognitive Load** already at 250 Hz → direct segmentation.
  - **SAM40** (native 128 Hz) : segmentation at 128 Hz, then each epoch resampled to 250 Hz using `scipy.signal.resample`.

#### Stress selection (SAM40)
- Reads `scales.xls` (columns `Subject No.`, `Trial_1`, `Trial_2`, `Trial_3` – maths scores).
- Keeps only subjects whose maximum score ≥ `STRESS_THRESHOLD` (default = 6).

#### Merging
- Concatenates epochs, labels (0/1), and subject IDs.
- Stress subject IDs are shifted by +15 to avoid collision with Concentration IDs (0‑14 → 15‑54).
- Saves the three arrays.

### Validation Results (post‑merge)

A separate validation script (not included) produced the following figures and metrics:

#### Label distribution
- Concentration : 1614 epochs (43.3%)
- Stress : 2112 epochs (56.7%)

#### Epochs per subject
- High variability due to different recording durations.

#### Spectral ratios (averages)

| Ratio | Formula | Concentration (mean) | Stress (mean) |
|-------|---------|----------------------|----------------|
| TBR (Theta/Beta) | θ (4‑8 Hz) / β (13‑30 Hz) | 6.688 | 2.455 |
| EI (Engagement Index) | β / (α + θ) | 0.395 | 0.426 |
| TAR (Theta/Alpha) | θ / α (8‑13 Hz) | – | – |

**Interpretation** :  
- **TBR** is much higher in concentration (6.7 vs 2.5), a known marker of cognitive load.  
- **EI** is nearly identical → possibly not discriminant on Fp2.  
- **TAR** values not shown but boxplots indicate separation.

#### Power Spectral Density (PSD)
- **Stress** shows pronounced peaks in β1 (13‑20 Hz) and α1 (8‑10 Hz).  
- **Concentration** has a flatter spectrum with moderate β1.

3. Filtre notch 50 Hz – rejet du bruit secteur.
4. Normalisation Z‑score `(x - μ)/σ` – rend les époques comparables.

### Segmentation
- Durée d’époque : 4 secondes (`EPOCH_S = 4.0`).
